1. The scope of 'var', 'let', and 'const' and there difference

Sajib01

In JavaScript, the scope of a variable determines where it can be accessed. There are three types of scope in JavaScript:

  • Global scope: Variables declared outside of any functions or blocks have global scope. This means they can be accessed from anywhere in the program.
  • Function scope: Variables declared inside a function have function scope. This means they can only be accessed from within the function that declared them.
  • Block scope: Variables declared inside a block (such as an if statement or a loop) have block scope. This means they can only be accessed from within the block that declared them.

2. The use cases of 'null' and 'undefined'

Sajib01

Null means an empty value or a blank value.Undefined means the variable has been declared, but its value has not been assigned.

  • Null: Null is basically an assignment value given to a variable. The variable which has been assigned as null contains no value. When we assign null as a value to any variable, it means that it is empty or blank. It is to show that the variable has no value. Also, null is an object in JavaScript. When it gets assigned to a variable, it represents no value.
  • Undefined: undefined means "not defined". So we declare a variable but do not assign a value to it, the variable becomes undefined. Unlike null, the value of an undefined variable is set by JavaScript as undefined. The variable gets created at the run-time. When we do not pass an argument for a function parameter, the default value is taken as undefined. Besides, when a function doesn't return a value, it returns undefined.

Example

let number = null;
console.log(number);
// output : null

let number;
console.log(number);
//output : undefined

3. What is REST API & how hey Work

Sajib01

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing web services. REST APIs are based on the HTTP protocol and use standard HTTP verbs (GET, POST, PUT, DELETE) to perform operations on resources.

  • How it works:REST APIs use standard HTTP methods to perform CRUD (Create, Read, Update, Delete) operations on these resources. The most commonly used HTTP methods are:

      Methods

    • GET: Used to retrieve data from the server, typically for reading or fetching resources.
    • POST: Used to create a new resource on the server.
    • PUT: Used to update an existing resource or create a new one if it doesn't exist.
    • PATCH: Used to partially update an existing resource
    • DELETE: Used to delete a resource from the server.
